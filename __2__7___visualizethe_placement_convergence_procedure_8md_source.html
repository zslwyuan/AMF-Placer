2 # Visualize the Placement Convergence Procedure
3 
4 If users set the "DumpAllCoordTrace" parameter in the JSON configuration file, a series of "DumpAllCoordTrace-XXX.gz" files which record the location of the design elements will be dumped to the specified directory. Each dumped archive file is for a lower-bound placement iteration or a upper-bound placement iteration in the global placement procedure.
5 
6 For evaluation and debugging, we provide Python script ("benchmarks/analysisScripts/paintPlacement.py") for users to visualize the placement trace with OpenGL. Please ensure that you have install OpenGL libs, if not you can install them by:
7 
8 ```
9 pip install PyOpenGL PyOpenGL_accelerate
10 ```
11 
12 Users can visualize the trace files by the following command:
13 
14 ```
15 usage example:
16 python paintPlacement.py -d xxxx/minimap2_allCellPinNet.zip -t xxxxx/dumpData_minimap_GENE -o xxxx/dumpData_minimap_GENE
17 ```
18 
19 1. -d indicate the design information archive, e.g. benchmarks/VCU108/design/minimap2/minimap2_allCellPinNet.zip
20 2. -t indicate the path where the placement trace archives are dumped. (the trace files are required to be named as DumpAllCoordTrace-xxx.gz currently)
21 3. -o indicate the path where you want to store the output images (png) generated according to the trace files
22
